# Contributing

Thanks for your interest in contributing to StrellerMinds Smart Contracts.

## Quick Links

- Detailed contribution guide: [docs/contributing.md](docs/contributing.md)
- Governance and triage: [.github/GOVERNANCE.md](.github/GOVERNANCE.md)
- Code of conduct: [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)

## Ways to Contribute

- Report bugs and security issues
- Improve contract correctness, performance, and test coverage
- Improve documentation and developer tooling
- Help triage issues and review pull requests

## Before You Start

1. Find or open an issue describing the work.
2. For large changes, discuss the approach in the issue first.
3. Follow the repo workflows in [docs/development.md](docs/development.md).

## Development Workflow

From the repository root:

```bash
make build
make unit-test
make check-code
```

## Pull Requests

- Keep PRs focused and linked to an issue.
- Add or update tests for behavioral changes.
- Update documentation when you change interfaces, events, or configuration.
- Ensure CI passes before requesting review.

## Community Expectations

All participation in this repository is governed by [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). By participating, you agree to uphold these standards in issues, discussions, reviews, and pull requests.
